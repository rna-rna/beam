{dragPosition && draggedItemIndex !== null && gallery?.images && (
  <motion.div
    className="fixed pointer-events-none z-50 ghost-image"
    style={{
      top: dragPosition.y,
      left: dragPosition.x,
      transform: "translate(-50%, -50%)",
      width: '100px',  // Reduced size
      height: '100px',  // Reduced size
      maxWidth: '10vw',  // Scale dynamically with viewport width
      maxHeight: '10vw',  // Maintain aspect ratio
    }}
    initial={{ opacity: 0, scale: 0.8 }}
    animate={{ 
      opacity: 0.9, 
      scale: 1,
      transition: {
        type: "spring",
        stiffness: 300,
        damping: 25
      }
    }}
    exit={{ opacity: 0, scale: 0.8 }}
  >
    <img
      src={gallery.images[draggedItemIndex].url}
      alt="Dragged Preview"
      className="w-full h-full object-cover rounded-lg shadow-lg"
    />
  </motion.div>
)}
Why This Fix Works:
By reducing the size to 100px and limiting the max dimensions to a percentage of viewport width (vw), the ghost image scales appropriately.
This keeps the preview unobtrusive but visible enough for accurate drag-and-drop placement.
